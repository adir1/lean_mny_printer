# LEAN Market Algorithm Analysis - Agent Instructions

## Project Overview

This is a **LEAN-based quantitative trading analysis project** built in Python. The goal is to analyze market algorithms for:
- **Alpha**: Excess returns generated by the strategy
- **Drawdown levels**: Peak-to-trough declines during trading periods
- **Risk metrics**: Sharpe ratio, Sortino ratio, maximum drawdown, volatility
- **Reward metrics**: Total returns, compound annual growth rate (CAGR), profit factor

## LEAN Engine Fundamentals

### Key Concepts
- **QCAlgorithm**: Base class for all LEAN strategies (see `main.py`)
- **Resolution**: Granularity of data (MINUTE, HOURLY, DAILY)
- **Slice**: Data container for each bar/tick event
- **Portfolio**: Tracks holdings, cash, and performance
- **Securities**: Individual tradable instruments (equities, futures, forex, crypto)

## Algorithm Structure

### Core Methods
- `initialize()`: Set date range, cash, data subscriptions, security settings
- `on_data(Slice data)`: Main event handler called on each new data point
- `on_end_of_algorithm()`: Called after backtest completes

### Common Patterns
```python
# Data subscription
self.add_equity(symbol, resolution)
self.add_future(symbol)
self.add_forex(pair)

# Portfolio management
self.set_holdings(symbol, percentage)
self.buy(symbol, quantity)
self.sell(symbol, quantity)

# Performance tracking
self.portfolio.invested  # bool
self.portfolio.cash      # float
self.portfolio.total_portfolio_value  # float
```

## Analysis Focus Areas

### 1. Alpha Calculation
- Compare strategy returns vs. benchmark (S&P 500)
- Use `Alpha` statistic from backtest results
- Positive alpha indicates outperformance

### 2. Drawdown Analysis
- **Maximum Drawdown**: Largest peak-to-trough decline
- **Drawdown Duration**: How long recovery takes
- Assess strategy resilience during market downturns

### 3. Risk Metrics
- **Volatility**: Standard deviation of returns
- **Sharpe Ratio**: Risk-adjusted returns (target annual: > 1.0)
- **Sortino Ratio**: Penalizes only downside volatility
- **Max Drawdown %**: Percentage decline from peak

### 4. Reward Metrics
- **Total Return %**: End-to-end strategy profitability
- **CAGR**: Compound annual growth rate
- **Profit Factor**: Gross profit / Gross loss (target: > 1.5)
- **Win Rate**: Percentage of profitable trades

## Configuration Files

### `config.json`
- `algorithm-language`: "Python"
- `parameters`: Strategy hyperparameters (can be optimized)
- `description`: Strategy documentation

### `main.py`
- Contains the `QCAlgorithm` subclass (e.g., `Leanmnyprinter`)
- Implements strategy logic in `initialize()` and `on_data()`
- Run locally with LEAN CLI or on QuantConnect platform

## Development Workflow

### Local Testing
1. Install LEAN CLI: https://www.quantconnect.com/docs/v2/lean-cli
2. Run algorithm: `lean backtester` or `lean live`
3. Review backtest statistics and logs

### Optimization
- Use `AddParameter()` in `initialize()` for hyperparameter tuning
- Run multiple backtests with different parameters
- Compare Sharpe ratio, max drawdown, and alpha metrics

### Debugging
- Use `self.debug(message)` to log insights
- Check logs for order fills, portfolio changes, errors
- Validate data subscriptions and resolution settings

## Code Standards for This Project

### Naming Conventions
- Class name: PascalCase (e.g., `Leanmnyprinter`)
- Methods: snake_case (e.g., `on_data()`, `initialize()`)
- Variables: snake_case (e.g., `portfolio_value`, `max_drawdown`)

### Best Practices
1. **Initialize once**: Set up all subscriptions and parameters in `initialize()`
2. **Event-driven**: Only trade logic belongs in `on_data()` — avoid loops over historical data
3. **Portfolio awareness**: Always check cash, holdings, and portfolio value before trading
4. **Documentation**: Comment complex logic and explain strategy rules
5. **Risk management**: Use stop losses, position sizing, and diversification

## Common Analysis Tasks

### Task: Calculate Alpha
- Fetch backtest statistics after run
- Compare strategy cumulative returns vs. benchmark
- Use `QCAlgorithm.Statistics` dictionary from results

### Task: Analyze Drawdown
- Extract daily portfolio values from backtest
- Calculate running maximum and drawdown percentage
- Identify dates of largest drawdowns

### Task: Optimize Strategy
- Add `AddParameter()` calls for tunable variables
- Run optimization across parameter ranges
- Rank by Sharpe ratio or risk-adjusted metrics

### Task: Add New Securities
- Use `add_equity()`, `add_future()`, `add_forex()`, `add_crypto()`
- Set `Resolution` appropriate to strategy timeframe
- Ensure data availability in LEAN data library
